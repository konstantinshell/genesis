# Промт: Расчёт Team MNPI

## Когда использовать
При обновлении командного дашборда — вручную или по расписанию.

## Что читать

1. `team/core/analysis.json` — состояние команды, личные MNPI (health.si, health.pi_avg, health.di, health.ri)
2. `partners/tasks.json` — список задач проекта (для расчёта TE)
3. `network/data.json` — сеть участников (продажи, рост бота для расчёта CF)

## Формула

```
Team MNPI = VB × TE × CF
```

### Уровень 1 — Витальная база (VB)

Среднее личных MNPI всех активных участников.

```
VB = (MNPI₁ + MNPI₂ + ... + MNPIₙ) / n
```

Личный MNPI = среднее из 4 компонентов: SI (сон), PI (феноменология), DI (десинхронизация), RI (ритмы).
Каждый компонент — от 0 до 10, нормализуется в 0–1.

**Правила:**
- Участник включается только после минимум 6 сессий MNPI
- Если ни у кого нет данных — VB = null, Team MNPI не считается
- Диапазон: 0.0 → 1.0

### Уровень 2 — Операционный КПД (TE)

```
TE = min(Tasks_Done / Tasks_Set, 1.2)
```

- `Tasks_Set` — общее количество задач в `partners/tasks.json`
- `Tasks_Done` — задачи со статусом `done`
- Если задач нет — TE = 0
- Диапазон: 0.0 → 1.2

### Уровень 3 — Коммерческий фактор (CF)

```
CF = 1 + (Sales_Actual / Sales_Target × 0.15) + (BotUsers_Growth_Rate × 0.10)
```

- `Sales_Actual` — количество оплативших в `network/data.json` (поле `paid: true`)
- `Sales_Target` — план продаж на период (по умолчанию 10 если не указан)
- `BotUsers_Growth_Rate` — прирост пользователей бота за последние 30 дней (0.20 = +20%)
- Диапазон: 1.0 → ~1.25

## Интерпретация

| Team MNPI | Статус | Цвет |
|-----------|--------|------|
| 0.85 – 1.0+ | Синергия | #34c759 (зелёный) |
| 0.70 – 0.84 | Рабочий режим | #daa040 (золотой) |
| 0.50 – 0.69 | Напряжение | #ff9f0a (оранжевый) |
| < 0.50 | Дисфункция | #ff453a (красный) |

## Формат ответа

Запиши результат в `team/core/team-mnpi.json`:

```json
{
  "value": 0.697,
  "vb": 0.72,
  "te": 0.85,
  "cf": 1.14,
  "status": "tension",
  "color": "#ff9f0a",
  "explanation": "VB = 0.72 — среднее личных MNPI двух участников (Константин 0.74, Натали 0.70). TE = 0.85 — выполнено 17 из 20 задач. CF = 1.14 — продажи на 80% от плана + рост бота 20%. Итог 0.697 — рабочий режим с потенциалом роста по всем уровням."
}
```

**Правила для explanation:**
- 2-3 предложения, конкретно
- Указать значения VB, TE, CF и что за ними стоит
- Подсветить слабое звено
- Без воды и похвалы

**Если данных нет:**
```json
{
  "value": null,
  "vb": null,
  "te": null,
  "cf": null,
  "status": "no_data",
  "color": "#555",
  "explanation": "Данные для расчёта Team MNPI ещё не собраны. Требуется: минимум 6 MNPI-сессий участников, задачи в потоке, данные по продажам."
}
```
